String源码分析

问题抛出：String是如何实现的？它有哪些重要的方法？

实现方法

以JDK1.8为例，String内部是以char数组实现的

构造函数

String有多个构造方法：

① 以String类型为参数的构造方法

② 以char数组类型为参数的构造方法，内部使用数组拷贝赋值给final类型的value数组

③ 以StringBuffer类型为参数的构造方法

④ 以StringBuilder类型为参数的构造方法

equals()方法

它重写了Object的equals()方法，当传入一个参数与字符串作比较时，先判断是不是自身调用equals()方法，如果是则直接返回true，否则继续判断参数类型是否为String类型，如果是则判断两个字符串的长度是否相等，如果相等则将两个字符串都转化为char[]数组，将每一位进行判断，如果遇到任意一位不相等都返回false。

与equals()方法相似的是equalsIgnoreCase()方法

compareTo()方法

compareTo()方法用于比较两个字符串是否相等，如果有任意一个字符不相等，则返回它们的差值

例如：1 - 2 = -1；1 - 1 = 0；2 - 1 = 1

还有一个相似的方法是compareToIgnoreCase()方法，忽略大小写进行比较

equals()和compareTo()的区别与联系

equals()和compareTo()都可以用于两个字符串的比较；

但是它们有两点不同：

① equals()方法可以接收Object类型的参数，也就是任意类型，但是compareTo()方法只可以接收String类型的参数；

② equals()方法的返回值是boolean，compareTo()方法的返回值是int

当equals()方法返回true时，或者compareTo()方法返回0时则代表两个字符串完全相等。

String其它重要方法

indexOf()：查询字符串首次出现的下标位置

lastIndexOf()：查询字符串最后出现的下标位置

contains()：查询字符串中是否包含另一个字符串

toLowerCase()：把字符串全部改成小写

toUpperCase()：把字符串全部改成大写

length()：返回字符串的长度

trim()：去掉字符串首尾空格

replace()：替换字符串中的某些字符

split()：把字符串分隔并返回字符串数组

join()：把字符串数组转为字符串

String问题延伸

为什么String类型要用final修饰？

用final修饰的类为不可变类，这样设计的好处是：Java语言之父James Gosling说过：他更倾向于使用final，因为它能够缓存结果，当你在传参时不用考虑谁会修改它的值，如果是可变类的话，则有可能要重新拷贝一个新值进行传参，在性能上有一定损失。另一个原因是安全，只有设计成不可变类，才能实现String的常量池缓存。

==和equals的区别是什么？

==对应基本数据类型来说是比较“值”是否相等的；而对于引用数据类型来说，是比较“对象引用地址”是否相等的。查看源码可以知道equals()方法是属于Object类型的，从String的equals()方法可以看出它重写了Object的equals()方法用于比较两个字符串的值是否相等。

String和StringBuffer、StringBuilder有什么区别？

StringBuffer提供了append()、insert()方法进行字符串的拼接，并使用synchronized保证线程安全。

由于线程安全，所以性能有所下降，所以在JDK1.5后出现了StringBuilder，它是非线程安全的，操作与StringBuffer相同，所以在非并发环境下考虑使用StringBuilder。

String的intern()方法有什么含义？

看两个创建String字符串的例子：

① String s1 = “java”;

② String s2 = new String(“java”);

它们在JVM的存储区域截然不同，字符串s1会先去字符串常量池中寻找是否含有java的字符串，如果找到则直接返回字符串常量池中的java，否则在常量池中创建一个java字符串对象，而s2使用new String("java")创建字符串对象时，不会往字符串常量池中创建java字符串对象，只有调用它的intern()方法时才会将java字符串放入常量池中。
