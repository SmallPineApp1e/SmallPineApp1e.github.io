进程的通信方式

为什么需要进程通信

每个用户进程都拥有各自不同的用户地址空间，任何一个进程的全局变量在另外一个进程中都无法看到，所以进程间通信必须要通过内核，在内核中开辟一块缓冲区，进程A把数据从用户空间拷贝到内存缓冲区，进程B再从内核缓冲区把数据读走，内核提供的这种机制叫做进程间通信。

进程间通信的本质

进程之间可以看到一份公共资源，提供这份资源的形式或者提供者不同，造成了通信方式不同，而pipe(管道)就是提供这份公共资源的形式之一。

管道通信

匿名管道

创建方式：调用pipe(int fd[2])函数创建管道，fd参数返回两个文件描述符，fd[0]指向管道的读端，fd[1]指向管道的写端，fd[1]的输入等于fd[0]的输出。

代码实现

管道通信的特点

  1. 管道只允许拥有血缘关系的进程间通信，例如父子进程间的通信。
  2. 管道只允许单向通信。
  3. 管道内部保证同步机制，从而保证访问的一致性
  4. 管道随进程的存在而存在，进程消失则管道对应的端口也消失，当两个管道都消亡时，管道也随之死亡。

信号量

信号量本质上是一个计数器（不设置全局变量是因为进程间是相互独立的，而这不一定能看到，看到也不能保证++引用计数为原子操作），用于多进程对共享数据对象的读取，它和管道有所不同，它不以传送数据为主要目的，它主要用来保护共享资源（信号量也是临界资源），使得资源在一个时刻只有一个进程独享。

信号量的工作原理

P(s)：将信号量s的值减一，如果结果大于等于0，则执行该进程；如果结果小于0，则挂起当前进程，进入阻塞状态等待，直到结果大于0

V(s)：将信号量s的值加1，如果结果小于等于0，则执行V操作的进程从信号量s有关的list等待队列中释放一个进程，使其转为就绪态，自己则继续执行，如果结果大于0，则执行V操作的进程继续执行，不需要做任何操作。

二元信号量

二元信号量（Binary Semaphore）是一种最简单的一种锁（互斥锁），它只用于两种状态：占用和非占用。所以它的引用计数为1

进程如何获得共享资源呢？

检测控制该进程的信号量

如果信号量的值大于0，则进程获得该共享资源的使用权，将信号量减1，代表它占用了一个资源单位

如果信号量的值小于等于0，则进入阻塞状态，直到信号量的值大于0时，如果进程被唤醒则返回至第1步

注明：信号量通过同步（信号量的PV操作满足原子性）和互斥（信号量的值）保证访问资源的一致性

共享内存

什么是共享内存

共享内存就是允许两个或多个进程共享一定的存储区。当一个进程改变了某块共享内存中的内容时，其它进程都会察觉到这个更改。因为数据不需要在客户机和服务器之间复制，数据直接写到内存当中去，不用若干次数据拷贝。

注明：共享内存没有任何的同步和互斥机制，也就是说，在第一个进程结束对共享内存的写操作之前，并无自动机制可以阻止第二个进程对它进行读取，所以要用信号量来实现对共享内存的存取和同步。

共享内存的速度最快

进程A给内存中写数据，进程B从内存中读取数据，在此期间发生了两次复制

（1）进程A到共享内存

（2）共享内存到进程B

因为直接在内存上操作，所以共享内存的速度就提高了

消息队列

消息队列提供了一种在两个不相关的进程之间传递数据的简单高效的方法

消息队列的特点

消息队列可以实现信息的随机查询，消息不一定以先进先出的次序读取，编程时可以按信息的类型读取

消息队列允许一个或多个进程向它写入或读取信息

与无名管道、命名管道一样，从消息队列中读出消息，消息队列中对应的数据都会被删除

每个消息队列都有消息队列标识符，消息队列的标识符在整个系统中是唯一的

消息队列是消息的链表，存放在内存中。
