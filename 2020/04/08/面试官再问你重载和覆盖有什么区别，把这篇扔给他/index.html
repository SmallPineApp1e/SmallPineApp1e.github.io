面试官再问你重载和覆盖有什么区别，把这篇扔给他

发表于 2020-04-08 | 分类于 Java
字数统计 1,299字 | 阅读时长 5分钟

面试官再问你重载和覆盖有什么区别，把这篇扔给他

如果面试官再问你重载和覆盖有什么关系？你一定要回答它，这两个概念就是名字比较像，思想是截然不同的！下面我会从实际代码来给你说明重载和覆盖的区别，并向你解释为什么会这样约束。

我们假定有三个类：Animal、Person、Dog，Dog覆盖了Animal的抽象方法，Person覆盖了Animal的抽象方法并重载了eat()方法，然后我们接着往下分析

覆盖父类方法的访问权限修饰符会在稍后说明，无需着急，我们先来看看Dog类和People类的eat()方法

覆盖（重写）

可以看到上面两个类中分别都有一个eat()方法，满足方法覆盖的要求是：两同两小一大

两同：形参列表相同、方法名相同

两小：子类返回值类型应该比父类返回值类型更小或相等，子类方法声明抛出的异常应该比父类更小或相等

一大：子类的访问权限应比父类方法的访问权限更大或相等

其中一大，从面向对象的思想来看，子类继承父类是为了具有父类暴露出来的行为和属性，在子类中应该至少保持与父类相同的暴露范围，例如eat()方法，在父类中的访问权限是protected，也就是任何继承于Animal的类都可以获得eat()这种行为，在子类Person或者Dog中，覆盖是重新定义这种行为，使行为具体化，所以Person和Dog的eat()方法表现出来的行为是不一样的，并且子类覆盖方法的作用范围至少应该保持与其父类的被继承的方法相同，或者具有更大的作用范围，所以eat()方法可以使用public和protected修饰符；Person和Dog的eat()方法行为的差异也印证了多态这种面向对象的思想。

重载

重载方法只要满足三个条件：在同一个类中、方法名相同、形参列表不同

重载只会发生在一个类中，而重写是父类和子类之间的行为，对于上面Person类中，我重载了eat()方法，传入了一个foodName属性，可以让人类吃任何东西，所以重载的目的是在同一个类中扩展同一个行为表现出来的结果，使该类的行为更加丰富，而覆盖的目的是让子类拥有父类的行为，并根据子类自身的特点进行行为的重定义，所以重载和重写是没有太大联系的，它们只不过是名字比较相像，容易混淆，所以面试官喜欢拿来问面试者，考察对这两个概念是否熟悉。

常问的面试题：为什么不可以根据方法的返回值类型来重载方法呢？

以Person类为例：

如果这样调用eat()方法，是没有问题的，因为它可以识别调用boolean返回值方法

但Java调用方法是可以忽略返回值的，如果这样调用方法，那么Java系统就会糊涂了，不知道你想调用哪个方法

所以Java重载方法不允许以返回值作为方法重载的标准。

结尾

我没有将重载和覆盖放在一起来比较，是因为我觉得它们本质上就是不同的两种思想，只要搞懂了重载是发生在同一个类中，覆盖是关于子父类的行为，那么对于重载和覆盖就不会混淆了，以上是个人对重载和覆盖的理解，如有错误，欢迎指出！虚心向大家请教学习！
